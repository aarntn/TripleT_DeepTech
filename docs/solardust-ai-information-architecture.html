SolarDust AI

User Flow, Feature Inventory, Page Structure, and Information Architecture

Frontend prototype reference for Figma refinement

Product positioning: SolarDust AI is a weather-aware solar farm operations dashboard that detects panel underperformance, separates dust-related losses from weather-related losses, recommends cleaning actions, estimates RM impact, and supports the business case for commercial O&M adoption.

1. Product Goal

The prototype should help judges and target O&M buyers understand one clear operational story: the system detects solar array performance loss, checks whether weather explains the loss, identifies likely dust accumulation, recommends cleaning only when useful, and quantifies the financial impact in RM.

Primary target users

  • Solar O&M manager: needs to prioritize cleaning actions and justify dispatch decisions.
  • Asset owner or portfolio manager: needs to protect energy yield and recurring revenue.
  • Procurement or commercial evaluator: needs ROI, payback, and pilot readiness evidence.
  • Field operations team: needs clear work order recommendations and selected array context.

Current target companies

  • Solarvest Holdings Bhd: asset owner and solar portfolio operator. Pitch angle: protect recurring asset income and reduce avoidable yield loss.
  • Pekat Teknologi: solar EPCC/O&M player expanding O&M capabilities. Pitch angle: add predictive cleaning intelligence to O&M offering.
  • TNB Energy Services: government-linked O&M pathway. Pitch angle: enterprise-ready decision support for renewable asset maintenance.

2. Current Navigation Structure

The current app uses a sidebar-based SaaS layout with two major contexts instead of many small pages.

Sidebar item Purpose User question answered
Panel Management Operations console for detecting, diagnosing, and acting on panel underperformance. Which panels are losing money, why, and what should we do now?
Business & ROI Commercial model for ROI, payback, carbon credits, tariff sensitivity, and dataset transparency. Is this worth buying, piloting, and scaling across a solar portfolio?

4. User Flow

Flow A: O&M manager handles panel underperformance

  1. User opens Panel Management.
  2. User scans top RM impact cards: RM lost today, RM lost this week, RM saved if cleaned now.
  3. User checks Farm A weather-aware operations summary.
  4. User reviews the map-style operations console and sees color-coded array zones.
  5. User clicks an array zone, for example B2.
  6. Selected array panel updates with efficiency, RM loss, sensor readings, classifier result, and AI dispatch insight.
  7. User checks whether the classifier says Dust, Weather, or Normal.
  8. If Dust, user creates a cleaning work order or uses Clean this panel.
  9. Dashboard animates cleaning, restores panel efficiency, and updates RM loss and saved value.
  10. User checks timeline to confirm expected vs actual output gap and forecast trend.

Flow B: Asset owner evaluates business impact

  1. User opens Business & ROI.
  2. User sets farm size in MW.
  3. User chooses target market, Malaysia or GCC desert.
  4. User adjusts tariff RM/kWh and cleaning system cost.
  5. User optionally activates Hormuz tariff shock.
  6. User reviews annual revenue recovered, payback period, NPV, and carbon credits.
  7. User reviews 12-month efficiency with cleaning vs without cleaning.
  8. User reviews cumulative savings vs system cost.
  9. User reviews water harvester synergy and monthly dataset assumptions.
  10. User uses the pitch sentence for procurement or pilot discussion.

5. Panel Management Page IA

Page purpose

Panel Management is the operational heart of the product. It should feel like a real solar farm command center, combining map context, sensor feed, weather forecast input, AI classification, recommendation, and cleaning action in one page.

Required content blocks

Section Content Design intent
Top RM impact cards RM lost today, RM lost this week, RM saved if cleaned now. Immediate business urgency. Judges should instantly understand financial value.
Weather-aware operations story Farm A live monitoring, forecast-driven explanation, dirty array count, selected array, selected status. Explain that weather forecast is an input, not a manual scenario toggle.
Farm operations map Map-style solar farm visualization with clickable A1, A2, B1, B2, C1, C2 zones. Make the dashboard feel like a real field operations product, not only analytics cards.
Left operations panel Farm metadata, active array count, irradiance, panel temperature, humidity, wind, forecast input, AI dispatch model. Give operational context similar to the reference image.
Floating selected-array detail Selected array name, status, efficiency, RM loss today, live sensor readings, classifier summary, AI dispatch insight, cleaning work order button. Support fast decisions after clicking a map zone.
Panel heatmap grid Six array cards with efficiency, status, RM loss, classifier signal, Clean this panel button, Clean all dirty panels button. Structured comparison view for all arrays.
Classifier card Dust, Weather, or Normal. Confidence score. One-line cause explanation. Highlight the core deep tech capability.
Recommendation card What is happening, recommended action, RM impact. Turn AI output into a plain-English maintenance decision.
Energy timeline chart Expected output, actual output, 3-day forecast, annotation for dust drop or weather event. Explain why the classifier made its decision.

Panel Management interactions

  • Clicking a map zone selects the array.
  • Clicking a heatmap card selects the array.
  • Selected array state updates map focus, selected detail panel, classifier card, recommendation card, and timeline chart.
  • Clean this panel creates a cleaning animation and restores the selected panel to high efficiency.
  • Clean all dirty panels cleans all dust-classified arrays.
  • Live sensor feed cycles every 3 seconds and updates sensor values, panel efficiency, panel color, and revenue stats.

Panel status rules

Efficiency Status Color
Above 90% Clean Green
60% to 90% Dust suspected Amber
Below 60% Heavy loss Red

8. Recommended Figma Refinement Notes

  • Keep the first visible screen focused on Panel Management, not Business & ROI.
  • Make the map view the hero of operations, with the RM impact cards above it.
  • Use the selected-array floating panel as the main interaction anchor.
  • Keep the classifier and recommendation cards visually important, but secondary to the map and selected array context.
  • For procurement viewers, make Business & ROI look calmer, more tabular, and more enterprise-ready.
  • Avoid adding too many separate pages. The current two-context structure is easier to demo.
  • Use clear labels: work order, cleaning priority, forecast input, dispatch insight, RM impact.
