AI Classifier Accuracy — Approach Options

How should the dashboard address the judge's "show me the model performance data" question?

A

Extend ClassifierCard with precision/recall table

Add a collapsible "Model Validation" section directly inside the existing ClassifierCard on Map View. Shows a small precision/recall/F1 table computed from the synthetic demo dataset already in the codebase. Framed honestly as "pre-pilot validation on synthetic data."

Pros

  • Minimal new code — extends existing component
  • Lives in the flow judges will see during demo
  • Shows real computed numbers, not just labels

Cons

  • Synthetic data — judges know it's not real-world
  • Confined to one panel at a time
B

New "Model Performance" card on Map View

Add a dedicated card below the panel list on Map View showing: model architecture summary (Random Forest + LSTM), a confusion matrix visualization, precision/recall/F1 from synthetic data, and a clear "validation roadmap" — what the paid pilot will produce.

Pros

  • Stands alone — can be pointed to directly
  • Shows full methodology story (architecture → metrics → next step)
  • Confusion matrix is visually impressive and shows rigor

Cons

  • More surface area to build
  • Sits below fold, needs deliberate navigation
C

New "AI Intelligence" sidebar page

Dedicated page under INSIGHTS in the sidebar. Full model story: architecture diagram, feature inputs (photodiode reflectance, Open-Meteo irradiance), precision/recall table, confusion matrix, benchmark citations (Nature 2025, PLOS ONE 2024), and a "Pilot Validation Plan" section.

Pros

  • Most comprehensive — directly pre-empts every sub-question
  • Can be navigated to during Q&A
  • Shows Cradle Fund-level technical depth

Cons

  • Largest build scope — Demo Day is 3 days away
  • New nav item might distract from main demo flow

My recommendation

Option B — new card on Map View. It's scoped to finish in a day, it lives where the demo naturally flows (judges will already be watching the classifier work), and the confusion matrix gives the "show me the data" moment without requiring a new nav page.
